# Audio Examples

`examples/audio` 目录包含 6 个音频示例，覆盖以下常见场景：

- 本地播放
- 网络播放
- 按键控制播放
- 录音
- 录音后回放

## Example Index

### `LocalPlay`

用途：播放保存在 LittleFS 中的本地音频文件。

适合场景：
- 验证本地音频文件能否正常播放
- 验证 LittleFS 文件路径是否正确
- 快速测试基础播放流程

### `URLPlay`

用途：连接 WiFi 后播放网络音频直链。

适合场景：
- 验证网络音频播放流程
- 测试 HTTP 音频直链播放
- 排查 WiFi 或网络流媒体相关问题

### `KeyControlPlay`

用途：播放本地音频文件，并在播放过程中通过实体按键和触摸按键控制：
- 暂停
- 恢复
- 停止
- 重新播放
- 音量加减

适合场景：
- 验证 `pause()`、`resume()` 和 `stop()` 接口行为
- 测试按键交互与播放状态切换
- 演示播放中的实时控制流程
- 演示触摸按键调节音量

### `Record`

用途：启动内部自动录音任务，录制固定时长后停止，
再回放录音结果。

适合场景：
- 验证最基础的自动录音能力
- 快速确认录音文件是否生成成功
- 快速试听录音效果

### `RecordBasic`

用途：使用默认录音参数，自动录音指定时长，
然后播放录下来的文件。

适合场景：
- 以最少参数体验录音 + 回放流程
- 作为录音接口的最简入门示例
- 快速验证默认参数是否满足需求

### `RecordAdvanced`

用途：显式设置采样率、位宽、声道数和播放音量，
再执行录音与回放。

适合场景：
- 测试不同录音参数组合
- 需要更明确地控制录音格式时
- 作为更完整的录音/播放配置示例

## Notes

- 本地文件和录音文件路径现在都可以不写前导 `/`，
  库内部会自动补齐。
- `play()` 会立即返回，播放状态可通过 `playing()`、`state()`
  和 `stateName()` 查询。
- 录音功能现在使用内部自动录音任务，
  不需要手动拉取音频数据。
- `KeyControlPlay` 默认使用：
  - A 键暂停/恢复
  - B 键停止
  - A+B 键重新播放
  - P 触摸键减小音量
  - Y 触摸键增大音量
